1. INTRODUCTION                                       *zeroxzero-intro*

0x0.nvim is a Neovim companion for the 0x0 AI coding assistant. This
plugin lets you send file and selection context from Neovim to a TUI
session's prompt stash, and pick which session to target.

The plugin connects to the 0x0-server daemon via HTTP and SSE for
file auto-reload events and statusline updates.

==============================================================================
2. SETUP                                              *zeroxzero-setup*

Requirements:
- Neovim >= 0.10
- `curl` in PATH
- `0x0-server` installed (https://github.com/anonymous-dev-org/0x0)
- `ANTHROPIC_API_KEY` env var set (or configured in config.yaml)

Using lazy.nvim: >lua
  {
    "anonymous-dev-org/0x0.nvim",
    opts = {},
  }
<

Using packer.nvim: >lua
  use {
    "anonymous-dev-org/0x0.nvim",
    config = function()
      require("zeroxzero").setup()
    end,
  }
<

Manual setup: >lua
  require("zeroxzero").setup({
    -- see |zeroxzero-config| for options
  })
<

==============================================================================
3. CONFIGURATION                                      *zeroxzero-config*

Default configuration: >lua
  require("zeroxzero").setup({
    cmd = "0x0-server",             -- Server binary (falls back to "0x0 serve")
    port = 4096,                    -- Server port
    hostname = "127.0.0.1",         -- Server hostname
    auto_start = true,              -- Start server if not running
    keymaps = {
      send = "<leader>0s",          -- Send file (n) or selection (v) to TUI
      switch_session = "<leader>0p",-- Switch pinned session
    },
    auth = nil,                     -- { username = "...", password = "..." }
  })
<

To disable a keymap, set it to empty string in setup.

==============================================================================
4. COMMANDS                                           *zeroxzero-commands*

  :ZeroSend                                           *:ZeroSend*
    Send current file reference to the TUI prompt stash.
    In visual mode, sends the selection with line references.

==============================================================================
5. KEYMAPS                                            *zeroxzero-keymaps*

Global keymaps (configurable via setup):

  <leader>0s     Normal      Send file reference to TUI
  <leader>0s     Visual      Send selection to TUI
  <leader>0p     Normal      Switch pinned session

==============================================================================
6. STATUSLINE                                         *zeroxzero-statusline*

Add 0x0 status to your statusline: >lua
  -- lualine example
  sections = {
    lualine_x = {
      { function() return require("zeroxzero").statusline() end },
    },
  }
<

Returns a string with connection status and current session title.
Returns empty string when not connected.

==============================================================================
7. HEALTH CHECK                                       *zeroxzero-health*

Run `:checkhealth zeroxzero` to verify your setup:
  - Neovim >= 0.10
  - curl available
  - 0x0-server binary found
  - Server connectivity
