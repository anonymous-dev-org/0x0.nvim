1. INTRODUCTION                                       *zeroxzero-intro*

0x0.nvim is a Neovim plugin that integrates with the 0x0 AI coding
assistant. It provides a native Neovim chat experience — a floating
window with streaming responses, foldable tool output, `vim.ui.input`
for prompts, and `vim.ui.select` for permissions. No terminal buffer.
Full vim motions and folds.

The plugin connects to the 0x0-server daemon via HTTP and SSE for
real-time message streaming, permission dialogs, question dialogs,
file reload events, and toast notifications.

==============================================================================
2. SETUP                                              *zeroxzero-setup*

Requirements:
- Neovim >= 0.10
- `curl` in PATH
- `0x0-server` installed (https://github.com/anonymous-dev-org/0x0)
- `ANTHROPIC_API_KEY` env var set (or configured in config.yaml)

Using lazy.nvim: >lua
  {
    "anonymous-dev-org/0x0.nvim",
    opts = {},
  }
<

Using packer.nvim: >lua
  use {
    "anonymous-dev-org/0x0.nvim",
    config = function()
      require("zeroxzero").setup()
    end,
  }
<

Manual setup: >lua
  require("zeroxzero").setup({
    -- see |zeroxzero-config| for options
  })
<

==============================================================================
3. CONFIGURATION                                      *zeroxzero-config*

Default configuration: >lua
  require("zeroxzero").setup({
    cmd = "0x0-server",             -- Server binary (falls back to "0x0 serve")
    port = 4096,                    -- Server port
    hostname = "127.0.0.1",         -- Server hostname
    auto_start = true,              -- Start server if not running
    chat = {
      width = 0.5,                  -- Fraction of editor width (0-1)
                                    -- or absolute columns (>1)
      height = 0.8,                 -- Fraction of editor height (0-1)
                                    -- or absolute rows (>1)
      border = "rounded",           -- Neovim border style
      fold_tools = true,            -- Auto-fold completed tool output
      show_thinking = false,        -- Show reasoning/thinking blocks
    },
    keymaps = {
      toggle = "<leader>0",         -- Toggle chat window
      context = "<leader>0f",       -- Add file (n) or selection (v)
      session = "<leader>0s",       -- Session picker
      interrupt = "<leader>0i",     -- Interrupt current response
      model = "<leader>0m",         -- Model picker
      inline_edit = "<leader>0e",   -- Inline edit
    },
    auth = nil,                     -- { username = "...", password = "..." }
  })
<

To disable a keymap, set it to empty string in setup.

==============================================================================
4. COMMANDS                                           *zeroxzero-commands*

  :ZeroToggle                                         *:ZeroToggle*
    Toggle chat window visibility (starts server if needed).

  :ZeroContext                                        *:ZeroContext*
    Add context to the next prompt. In normal mode, adds the
    current file. In visual mode, adds the selection.

  :ZeroSession                                        *:ZeroSession*
    Open session picker. Lists existing sessions and a
    "New session" option. Creates a new session directly
    if no sessions exist.

  :ZeroInterrupt                                      *:ZeroInterrupt*
    Interrupt the current response.

  :ZeroModel                                          *:ZeroModel*
    Open model picker (lists all providers and models).

  :ZeroInlineEdit                                     *:ZeroInlineEdit*
    Inline edit at cursor position or visual selection.
    Creates a temporary session, sends the edit instruction,
    and auto-reloads the file when done.

==============================================================================
5. KEYMAPS                                            *zeroxzero-keymaps*

Global keymaps (configurable via setup):

  <leader>0      Normal      Toggle chat window
  <leader>0f     Normal      Add current file to context
  <leader>0f     Visual      Add selection to context
  <leader>0s     Normal      Session picker
  <leader>0i     Normal      Interrupt current response
  <leader>0m     Normal      Model picker
  <leader>0e     Normal/Vis  Inline edit

Chat buffer keymaps (hardcoded):

  q              Close chat window
  <CR>           New prompt
  <C-c>          Interrupt current response
  <Tab>          Cycle to next agent (per-prompt)
  <S-Tab>        Cycle to previous agent (per-prompt)
  za             Toggle fold (tool output)
  zM             Fold all
  zR             Unfold all

==============================================================================
6. CHAT WINDOW                                        *zeroxzero-chat*

The chat window is a native Neovim floating buffer with:

- Streaming responses — text appears as the model generates it
- Foldable tool output — tool invocations fold into single lines
  with status icons (✔ success, ✘ error, ▶ running)
- Permission dialogs — vim.ui.select for allow/reject decisions
- Question dialogs — multi-step vim.ui.select/vim.ui.input
- Auto-scroll — follows output unless you scroll up
- Session switching — load history from any session

Chat buffer format: >

  ────────────────────────────────────────────────────
   You
  ────────────────────────────────────────────────────
  fix the bug in auth.ts

  ────────────────────────────────────────────────────
   Assistant (claude-sonnet-4-6)
  ────────────────────────────────────────────────────

  I'll fix the authentication bug.

  ✔ edit src/auth.ts [+5/-3]
  │ @@ -10,5 +10,7 @@
  │  function auth() {
  │ -  return false
  │ +  return true
  │  }

  ✔ bash npm test
  │ PASS src/auth.test.ts

  The tests pass now.
<

Tool output lines prefixed with │ are foldable. When fold_tools
is enabled (default), completed tool blocks auto-fold.

==============================================================================
7. STATUSLINE                                         *zeroxzero-statusline*

Add 0x0 status to your statusline: >lua
  -- lualine example
  sections = {
    lualine_x = {
      { function() return require("zeroxzero").statusline() end },
    },
  }
<

Returns a string with connection status, animated spinner when busy,
and current session title. Returns empty string when not connected.
